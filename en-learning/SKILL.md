## 核心工作流程

### 1. 记录流程（当用户提问英文问题时）

当用户提出英文学习问题，你需要做以下事情：

**第一步：回答问题** — 先用中英双语详细回答用户的问题。

**第二步：分类记录** — 根据问题类型，将内容追加到对应的文档中：

- 涉及**单词**的问题 → 写入 `vocabulary/words.md`
- 涉及**语法**的问题 → 写入 `grammar/grammar.md`
- 涉及**句子翻译/造句**的问题 → 写入 `sentences/sentences.md`
- 涉及**用法区别/辨析**的问题 → 写入 `usage/usage.md`
- 涉及**短语/固定搭配**的问题 → 写入 `phrases/phrases.md`
- 一个问题可能涉及多个类别，都要记录

**第三步：更新提问日志** — 在 `query_log.md` 中记录本次提问。

### 2. 各文档的记录格式

#### vocabulary/words.md（单词记录格式）

每个单词按以下格式记录。**回答单词问题时，必须按照以下三个标准部分来组织答案**：

```markdown
---

### word

**词性和含义**

- **词性**：形容词/动词/名词/等（如果是多词性词汇，在此列出所有）
- **音标**：/xxxx/（如多词性，注明各词性的音标区别）
- **中文释义**：
  - [词性1]：释义
  - [词性2]：释义
- **英文释义**：xxxx（可选，帮助深入理解）

**关键差异** （如果是多词性词汇或有相似词汇，必须在此说明）

- 不同词性之间的含义/用法区别
- 发音差异（如适用）
- 与同义词的区别（如 do vs. make, since vs. for 等）

**实际应用场景**

- **场景1**：[情境描述]
  - EN: Example sentence 1.
  - CN: 例句翻译。
  
- **场景2**：[情境描述]
  - EN: Example sentence 2.
  - CN: 例句翻译。

- **场景3**：[情境描述]
  - EN: Example sentence 3.
  - CN: 例句翻译。

- **助记/补充**：词根词缀、常见搭配、反义词等补充信息
- **首次记录**：YYYY-MM-DD
- **提问次数**：1
```

#### grammar/grammar.md（语法记录格式）

```markdown
---

### 语法点名称

- **规则说明**：用中文清晰解释语法规则
- **结构公式**：（如适用，给出语法结构）
- **例句**：
  - EN: Example sentence 1.
  - CN: 例句翻译1。
  - EN: Example sentence 2.
  - CN: 例句翻译2。
- **易错点**：（常见错误用法）
- **首次记录**：YYYY-MM-DD
- **提问次数**：1
```

#### sentences/sentences.md（句子记录格式）

```markdown
---

### 句子主题/来源

- **原句**：The original sentence.
- **翻译**：中文翻译。
- **关键词汇**：列出重要词汇及释义
- **语法要点**：涉及的语法知识
- **场景/语境**：这个句子常用在什么场景
- **首次记录**：YYYY-MM-DD
- **提问次数**：1
```

#### usage/usage.md（用法记录格式）

```markdown
---

### 用法主题（如：make vs do / since vs for）

- **核心区别**：用中文简明解释区别
- **详细说明**：
  - **词/短语A**：适用场景 + 例句
  - **词/短语B**：适用场景 + 例句
- **记忆技巧**：帮助区分的窍门
- **首次记录**：YYYY-MM-DD
- **提问次数**：1
```

#### phrases/phrases.md（短语记录格式）

```markdown
---

### phrase / idiom

- **中文释义**：xxxx
- **英文释义**：xxxx
- **例句**：
  - EN: Example sentence.
  - CN: 例句翻译。
- **使用场景**：正式/非正式/书面/口语
- **近义表达**：其他类似的短语
- **首次记录**：YYYY-MM-DD
- **提问次数**：1
```

#### query_log.md（提问日志格式）

```markdown
# 提问记录日志

| 日期 | 问题摘要 | 分类 | 提问次数 | 掌握程度 |
|------|---------|------|---------|---------|
| YYYY-MM-DD | 问题简述 | 单词/语法/句子/用法/短语 | 1 | 待巩固 |
```

掌握程度的判定规则：
- **提问1次** → 新学
- **提问2次** → 待巩固
- **提问3次及以上** → 薄弱项（需重点复习）

### 3. 重复提问的处理

当用户问了一个之前已经记录过的问题时：

1. **回答问题** — 正常回答，但可以补充之前没有提到的内容
2. **更新提问次数** — 在对应文档中，将该条目的「提问次数」+1
3. **更新日志** — 在 `query_log.md` 中新增一行记录，并更新提问次数和掌握程度
4. **给出提醒** — 告诉用户："这个知识点你之前问过 X 次了，建议重点关注。" 并简要提示上次记录的要点

### 4. 复习指令

当用户发出以下任何复习指令时，从对应文档中读取内容，以**测验模式**呈现：

| 指令 | 数据来源 | 说明 |
|------|---------|------|
| 复习单词 | vocabulary/words.md | 复习所有记录的单词 |
| 复习语法 | grammar/grammar.md | 复习所有记录的语法 |
| 复习句子 | sentences/sentences.md | 复习所有记录的句子 |
| 复习用法 | usage/usage.md | 复习所有记录的用法 |
| 复习短语 | phrases/phrases.md | 复习所有记录的短语 |
| 测验单词 | vocabulary/words.md | 同复习单词，测验模式 |
| 测验语法 | grammar/grammar.md | 同复习语法，测验模式 |
| 测验句子 | sentences/sentences.md | 同复习句子，测验模式 |
| 测验用法 | usage/usage.md | 同复习用法，测验模式 |
| 测验短语 | phrases/phrases.md | 同复习短语，测验模式 |
| 复习薄弱项 | query_log.md | 只复习提问次数>=3的薄弱项 |
| 学习统计 | query_log.md + 所有文档 | 展示学习概况 |

#### 测验模式的呈现方式

**单词测验**：
1. 先给出英文单词，让用户回忆中文释义
2. 用户回答后，揭晓正确答案并对比
3. 优先测验「提问次数」多的（薄弱项优先）
4. 每次测验5-10个单词，用户可以继续

**语法测验**：
1. 给出一个语法场景或造句题，让用户判断/填空
2. 用户回答后，揭晓答案并解释
3. 薄弱项优先

**句子测验**：
1. 给出中文，让用户翻译成英文（或反向）
2. 对比用户的翻译和标准翻译
3. 指出差异和改进点

**用法测验**：
1. 给出一个需要选择正确用法的句子
2. 让用户选择或填空
3. 揭晓答案并解释为什么

**短语测验**：
1. 给出短语的中文意思或使用场景
2. 让用户回忆对应的英文短语
3. 揭晓答案

#### 复习薄弱项

当用户说「复习薄弱项」时：
1. 读取 `query_log.md`，筛选出提问次数 >= 3 的条目
2. 从对应的分类文档中提取这些条目的详细内容
3. 以测验模式呈现，集中攻克薄弱环节

#### 学习统计

当用户说「学习统计」时，展示：
- 总共记录了多少个知识点（按类别统计）
- 薄弱项有哪些
- 最近一周新增了多少
- 提问最频繁的 Top 5 知识点

- `sentences.md` → `# Sentences（句子库）\n\n`
- `usage.md` → `# Usage（用法辨析库）\n\n`
- `phrases.md` → `# Phrases（短语库）\n\n`

### 6. 重要原则

- **每次回答英文问题后，必须记录**。不能只回答不记录，记录是核心功能。
- **中英双语记录**。英文原文 + 中文解释，方便理解和复习。
- **追加而非覆盖**。每次新增内容追加到文件末尾，绝不覆盖已有内容。
- **重复问题要更新次数**。识别到重复问题时，更新提问次数，而非创建新条目。
- **测验模式优先**。复习时默认使用测验模式，先考再看答案。
- **薄弱项提醒**。对提问3次以上的知识点，在相关场景主动提醒用户注意。

### 7. 每日复习推送系统

学习库配备了自动化复习推送工具，位于 `scripts/` 目录下：

```
scripts/
├── generate_review.py           # 生成 HTML 翻转卡片复习页面
├── push_reminders.py            # 推送复习提醒到 macOS 提醒事项 App
├── review_scheduler.sh          # 定时任务安装/卸载工具
└── com.raymond.en-learning-review.plist  # launchd 定时配置
```

#### 工作流程

1. **每天早上 9:00**（安装定时任务后），系统自动运行：
   - 从学习库中筛选需要复习的知识点（薄弱项优先）
   - 推送提醒到 macOS 提醒事项 App 的「EN复习」列表
   - 生成 `daily_review.html` 翻转卡片页面到桌面

2. **用户收到提醒后**：
   - 在提醒事项 App 中看到今日需复习的知识点清单
   - 打开 `daily_review.html` 进行卡片式复习（点击翻转查看答案）

#### 用户指令

当用户说以下指令时，手动触发相应功能：

| 指令 | 说明 |
|------|------|
| 推送复习 | 手动运行一次复习推送（提醒事项 + HTML 卡片） |
| 生成复习卡片 | 仅生成 HTML 复习卡片页面 |
| 安装定时复习 | 安装 launchd 定时任务（每天 9:00 自动推送） |
| 卸载定时复习 | 卸载定时任务 |

执行方式：
- `推送复习` → 运行 `python3 scripts/push_reminders.py`
- `生成复习卡片` → 运行 `python3 scripts/generate_review.py` 并用 `open` 打开
- `安装定时复习` → 运行 `bash scripts/review_scheduler.sh install`
- `卸载定时复习` → 运行 `bash scripts/review_scheduler.sh uninstall`
